##docker监控

我是用的是SuitAgent，一个第三方插件。

直接clone最新的代码或者下载zip包进行安装

or我的云盘共享地址

SuitAgent文档

下载完成后自行编译，此时需要安装Java环境和maven

Java1.8 Linux64存放地址

maven3.5存放地址

解压后配置环境变量即可使用

####设置环境变量：

vi /etc/profile
export M2_HOME=/home/ubuntu/apache-maven-3.5.0
export JAVA_HOME=/home/ubuntu/jdk1.8.0_144
export CLASSPATH=.:$JAVA_HOME/lib:$CLASSPATH
export PATH=$JAVA_HOME/bin:$JAVA_HOME/jre/bin:$M2_HOME/bin:$PATH

source /etc/profile

####编译

选择对应的版本进行编译，我们机器是linux-64，所以选择如下第一条指令在下载的suitAgent-master根目录下执行即可开始编译

  • linux-64 : mvn clean package -Plinux-64 -Dmaven.test.skip=true
  • linux64-noJar : mvn clean package -Plinux64-noJar -Dmaven.test.skip=true

  • osx-noJar : mvn clean package -Posx-noJar -Dmaven.test.skip=true

    注意，此时编译会报错，缺少oracle的依赖。

    Oracle依赖包下载地址：http://pan.baidu.com/s/1mhIzn5I

    下载依赖包之后，解压后将repository与maven的仓库进行合并，默认的maven仓库地址在~/.m2

    装好依赖之后，然后我们再重新进行编译。

    编译成功后生成的编译文件存在target文件夹。

    cd target
    tar -zxvf falcon-agent-linux64.tar.gz
    cd falcon-agent
    vim SuitAgent-master/target/falcon-agent/conf/falcon/agent.cfg.json
    修改heartbeat和transfer的地址，正确的地址应该是push到的监控系统的地址

    cd SuitAgent-master/target/falcon-agent
    sudo ./bin/agent.sh start
    sudo ./bin/agent.sh status

启动没有问题就完成了！然后登陆监控系统后台dashbord应该就能收到推送的信息了。
